	Software Test Plan reprezinta un plan ce detaliaza:-scopul testarii;
							   -planificarea in timp;
							   -cerintele ce se testeaza.
	Pentru testarea manuala va trebui sa impartim aplicatia de inscriere la licenta in mai multe componente:index, login, register, user_dashboard, account_manager, account_manager_user, admin_dashboard, create_new_account, edit_accounts, edit_comittees, edit_license, exam_rules, insert_license_info. Urmand apoi sa testeam fiecare componenta in parte si de a efectua cate un raport de testing si o diagrama pentru fiecare. Se va verifica corecta functionare a fiecarei compenente in parte dar si a intregului ansamblu, urmand a se pune accentul pe situatiile in care ar putea aparea probleme de exemplu pentru componenta register o problema ar fi atunci cand un user nou ar avea acelasi username si aceelasi parola un un user deja inregistrat.Se vor testa componentele in functie de de responsiveness dar si validarea si corectitudinea datelor ce sunt introduse sau afisate pe interfata de comunica cu utilizatorul deoarece acest lucru este este unul foarte important pentru componenta de frontend.
